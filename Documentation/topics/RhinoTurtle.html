Rhino Turtle Commands

General Notes

The Turtle is passed arguments that are floating point values, but the examples show unprocessed arguments (before they are evaluated). All arguments provided in the LSystem file are processed through the Expression evaluator. All defined variables are accessible. Where integer values are expected, values are checked for validity: 1.0 equals 1, but 1.1 does not equal 1 and will throw an error. The examples omit the LSystem file statement " alias : < letter > " , but do remember to include this in your definition files.

The Modeling Process

The LSystem Turtle for Rhino is a stack based modeller. There are two stacks inside the Turtle: one for points and one for curves. Stacks are intialized with "push" calls which intializes a list for either points or curves. Points and Curves are added to the lists, and then the list is "popped" off the stack and geometry is created with the list. What geometry is created is a function of the specific Turtle command. Points can be used to create curves and meshes, while Curves can be used to create surfaces. Stacks can be nested, where lists can be created on top of each other. This deferes "pop" operations on lists. For the point stack it would look like this: "{....{....}}" Where "{" = push, "}" = pop, and "." = drop point (records Turtle's position). The example would create two curves, but the second one, since it is "popped" first, is created first. Different Turtle commands will require different numbers of points or curves, refer to the individual Turtle commands for specifics.

Environment

SetTropismActive : < switch >

Turns Tropism on. This is global for the LSystem and any nested LSystems. The turtle's X Axis is considered the heading and is the axis attracted to the Tropism vector. Currently, this does not produce the same results as ABOP, it seems much stronger. Setting example multipliers (TropismFactor) down produces better results.

< switch > = 1 for on, 0 for off

#turns tropism on
SetTropismActive : 1

SetTropismFactor : < factor >

Sets the turtle's susceptibility to bending by the Tropism vector.

< factor > = 0 to 1 for turtle's bending susceptibility

#set the tropism factor
SetTropismFactor : 0.23

SetTropismVector : < x > : < y > : < z >

Sets the Tropism vector. This is the vector that the turtle aligns itself to based on the Tropism Factor.

< x > = Tropism vector x component
< y > = Tropism vector y component
< z > = Tropism vector z component

#set the tropism vector
SetTropismVector : 0.1 : 0.1 : -1.2

Transforms and Alignment

MoveOnAxis : < axis > : < distance >

Moves the Turtle along it's local axis.

< axis > = 0 for x axis, 1 for y axis, 2 for z axis
< distance > = distance to move on axis

#move along x axis (heading) by value of variable "dist"
MoveOnAxis : 0 : dist
#move (laterally) on y axis by 10
MoveOnAxis : 1 : 10

MoveInWorld : < x > : < y > : < z >

Moves the Turtle in world coordinates (relative move).

< x > = distance to move in x
< y > = distance to move in y
< z > = distance to move in z

#move in world coordinates by 3 for x, y, & z
MoveInWorld : 3 : 3 : 3

MoveOnPlane : < plane > : < x > : < y > : < z >

Moves the Turtle locally in the plane specified. Turtle orientation is preserved.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< x > = distance to move in x
< y > = distance to move in y
< z > = distance to move in z

#move locally to xy plane 3, 5, & 7 units in x,y,z
MoveOnPlane : 0 : 3 : 5 : 7
#move locally to yz plane 3, 5, & 0 units in x,y,z - this is the quivelent of moving 0,3,5 in the world xy plane
MoveOnPlane : 1 : 3 : 5 : 0

RotateOnAxis : < axis > : < radians >

Rotates the Turtle around it's local axis.

< axis > = 0 for x axis, 1 for y axis, 2 for z axis
< radians > = The amount to rotate the Turtle in radians.

#rotate around the z axis 45 degrees
RotateOnAxis : 2 : rad(45)

AlignToPlane : < plane >

Aligns the Turtle to the specified plane. Turtle location is preserved.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane

#aligns Turtle to XY Plane
AlignToPlane : 0

RollToLevel

Rotates the Y Axis around the X Axis until the Y Axis lays in the World XY Plane. Turtle location is preserved. This is the '$' command described in ABOP on Page 57 (Figure 2.6, page 56). This command takes no arguments.

#rolls the Turtle to level
RollToLevel

Contents

Stack

PushStack

Records the current Turtle state (position, orientation, material index, etc) and pushes onto the record stack for retrieval later. This command takes no arguments.

PushStack

PopStack

Pops a Turtle state record off the stack, and sets the Turtle back to that state (position, orientation, material index, etc). This command takes no arguments.

PopStack

PruneBranch

Prunes the rest of the branch. After this command, no Turtle commands will be issued until the next PopStack (end of branch) is encountered. This command takes no arguments.

PruneBranch

Curves

MoveOnAxisWithLine : < axis > : < length >

Moves the Turtle along it's local axis and creates a line from it's begining and end position. This command is provided as an equivalent to "F" (go with pen down) described in ABOP.

< axis > = 0 for x axis, 1 for y axis, 2 for z axis
< length > = length of the line to draw and distance to move

#draws a line along the x axis (heading) of the length set in variable "dist"
Line : 0 : dist

Line : < plane > : < destination > : < x > : < y > : < z >

Creates a line from the Turtle's current position to the coordinates in the local plane space specified. The plane is defined locally to the Turtle, meaning that orientation in world space is preserved; i.e.: (the YZ Plane is constructed from the turtle's current y and z axis). The Turtle is not moved in the command. The destination controls where the resulting Line is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< x > = x component of line end point
< y > = y component of line end point
< z > = z component of line end point

#draws a line in the local xy plane,
Line : 0 : 0 : 3 : 3 : 3

Arc : < plane > : < destination > : < radius > : < start angle > : < end angle >

Creates an arc in the plane specified centered at the Turtle's position and drawn at the radius specified, from the start angle to the end angle. The plane is defined locally to the Turtle, meaning that orientation in world space is preserved; i.e.: (the YZ Plane is constructed from the turtle's current y and z axis). The Turtle is not moved in the command. The destination controls where the resulting Arc is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< major radius > = major radius of the ellipse (x axis)
< minor radius > = minor radius of the ellipse (y axis)

#draws an arc on the local xy plane with radius of 1, from 30 to 270 degrees
#the arc is created in the current document
Arc : 0 : 1 : 10 : rad(30) : rad(270)

Circle : < plane > : < destination > : < radius >

Creates a circle in the plane specified, centered at the Turtle's position using the radius provided. The plane is defined locally to the Turtle, meaning that orientation in world space is preserved; i.e.: (the YZ Plane is constructed from the turtle's current y and z axis). The Turtle is not moved in the command. The destination controls where the resulting Circle is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< radius > = radius of the circle

#draws a circle on the local xy plane with a radius set in variable "radius"
#the circle is drawn in the document and not placed in the curve stack Circle : 0 : 1 : radius

Ellipse : < plane > : < destination > : < major radius > : < minor radius >

Creates an ellipse in the plane specified, centered at the Turtle's position using the major and minor radii provided. The plane is defined locally to the Turtle, meaning that orientation in world space is preserved; i.e.: (the YZ Plane is constructed from the turtle's current y and z axis). The Turtle is not moved in the command. The destination controls where the resulting Ellipse is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< major radius > = major radius of the ellipse (x axis)
< minor radius > = minor radius of the ellipse (y axis)

#draws an ellipse on the local xy plane with radii set in variables "r1" and "r2"
#the ellipse is pushed to the curve stack
Ellipse : 0 : 0 : r1 : r2

Polygon : < plane > : < destination > : < radius > : < sides >

Creates a polygon in the plane specified. The plane is defined locally to the Turtle, meaning that orientation in world space is preserved; i.e.: (the YZ Plane is constructed from the turtle's current y and z axis). The polygon is centered at the Turtle's position and is drawn inscribed in the radius specified with the number of sides specified. The Turtle is not moved in the command. The destination controls where the resulting Arc is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< radius > = major radius of the ellipse (x axis)

#draws a polygon on the local xy plane with radius of 5 with 7 sides
#the arc is created in the current document
Polygon : 0 : 1 : 5 : 7

Point Stack

PushPointStack

Pushes a new point list onto the point stack. This must always be called before you can append points to the point list.

PushPointStack

Point : < destination > : < front >

Records the Turtle's position in the current point list and or creates a Point in the document. The flag "front" is a boolean value used when adding points to the Point Stack. It must be provided even if you are not creating a point on the Point Stack (it is ignored if the destination is document). Remember that you are responsible for intializing the Point Stack before creating points there.

< destination > = 0 for Point Stack, 1 for Document, 2 for both
< front > = boolean: true= point is added to front of point list, false=end of point list (this flag must always be provided)

# create a point in the document using pre-defined constants for flags
Point : D_DOC : FALSE

PopPointsToCurve : < destination > : < close >

Takes the current point list and builds a curve from the points. A list of 2 points will generate a Line. A list of 3 points will generate a three-point Arc (start, point on cuvrve, end). If the list has more than 3 points, an interpolated curve is generated (3rd degree). You can also create closed curves. If a closed curve is specified for 3 points a circle is created instead of an arc. The destination controls where the resulting Curve is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< close > = 0 for open curve, 1 for closed curve

#creates and open curve in document and push the curve to curve stack
PopPointsToCurve : 2 : 0

PopPointsToPolyline : < destination > : < close >

Takes the current point list and builds a polyline from the points. The destination controls where the resulting Curve is put: the Curve Stack, the Rhino Document, or both. Remember that you are still responsible for calling PushCurveStack before any curves are placed on the stack.

< destination > = 0 for Curve Stack, 1 for Document, 2 for both
< close > = 0 for open curve, 1 for closed curve

#create a closed polyline on the curve stack
PopPointsToPolyline : 0 : 1

PopPointsToPolygons

*** This is currently a work in progress - Three point faces work well. ***
Takes the current point list and builds a mesh from the points. The point list must contain at least 3 points to generate a mesh. Currently, a best fit planar texture mapping is applied. This command takes no arguments.

#create a mesh from the current point stack
PopPointsToPolygons

PopPointsToMeshBorder

This is the equivalent to the Rhino Command for creating a mesh from a closed polyline. The points are used to create a closed polyline which is used as the mesh border. Currently, a best fit planar texture mapping is applied. This command takes no arguments.

#create a mesh from a closed polyline built from the current point stack
PopPointsToMeshBorder

Curve Stack

PushCurveStack

Creates a new Curve List on the Curve Stack. This command takes no arguments.

#push new curve list to stack
PushCurveStack

PopCurveStackToLoft : < loft type > : < normalize seam > :

Pops the current curve list and builds a lofted surface from the curves. There must be at least two curve in the Curve List. You can use normalize seam to unify the indvidual curve's seam points with minimal twisting. This operation will take longer.

< loft type > = integer value for loft type: normal = 0, loose = 1, tight = 2, straight = 3, developable = 4, uniform = 5
< normalize > = 1 true, 0 for false: normalizes the surface seam

#create a normal style loft from the curves on the stack with a normalized seam
PopCurveStackToLoft : L_NORMAL : TRUE

PopCurveStackToSweep

Pops the current curve list and builds a sweep surface with one or two rails from the curves. If there are two curves on the stack, then first is the rail and the second is the shape. If there are three curves on the stack, then first and second are the rails and the second is the shape. This command takes no arguments.

#create sweep surface
PopCurveStackToSweep

PopCurveStackToRevolve : < axis > : < start angle > : < end angle >

Pops the current curve list and builds a revolve surface with it by rotating around a local access from start to end angle. There can be only one curve on the stack.

< axis > = 0 for x axis, 1 for y axis, 2 for z axis
< start angle > = start angle of revolution
< end angle > = end angle of revolution

#create a revolve surface by revolving a curve around z axis from 45-135 degrees
PopCurveStackToRevolve : 2 : rad(45) : rad(135)

PopCurveStackToExtrude : < height > : < cap >

Pops the current curve list and extrudes the curve. There should be only one curve in the curve list. You can cap the extrude to create a solid form.

< height > = height of the extrusion
< cap > = create cap: 1 = caps, 0 = no caps

#extrudes the curve in the current curve list stack
#shape is 5 units high with caps
PopCurveStackToExtrude : 5 : 1

Blocks

InsertBlock : < plane > : < id > : < scale x > : < scale y > : < scale z >

Inserts a block reference into the model oriented to the local Turtle plane specified. The Turtle's position is used as the insertion point, and the block is scaled according to the x,y,z scale inputs. The block name is composed of the prefix: "LSystemBlockRef" with 'id' (an integer value) appended to the end. I.e.: "LSystemBlockRef1", "LSystemBlockRef45106". If the block does not exit in the model, it will be created so you may redefine the geometry after the LSystem has executed.

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< id > = integer value to append to name prefix forming the block name
< scale x > = x scale factor of block
< scale y > = y scale factor of block
< scale z > = z scale factor of block

#insert block LSystemBlockRef33 in the local xy plane
# scale block by 1,4,2 in x,y,z
InsertBlock : 0 : 33 : 1 : 4 : 2

Parametric Objects

BranchSection : < plane > : < radius > : < randomness > : < detail >

Experimental parametric object designed for use with loft surfaces. This creates a closed curve with randomess directly into the Turtle's curve stack. The curve is basically a circle with randomness applied. The amount of detail (number of points) determines the final shape based on the randomness. Remember that you are responsible for creating a curve list before this is called (PushCurveStack).

< plane > = 0 for xy plane, 1 for yz plane, 2 for xz plane
< radius > = rough radius of closed curve
< randomness > = randomness of closed curve from 0 to 1
< detail > = number of points in curve, min=3

#create a branch section on the curve stack
# section has a rough radius of 2 with 0.2 randomness and five control points
BranchSection : 0 : 2 : 0.2 : 5

StraightTube : < radius > : < length > : < radius factor > : < cap >

Creates a straight tube with a taper and optional half sphere cap.

< radius > = radius of tube in radians at base
< length > = length of tube
< radius factor > = multiplier for end radius of tube (scales radius)
< cap > = creates semi-spherical cap: 1=cap, 0=no cap

# create a tube with a radius of 2 and length of dist
#tube will be straight since factor is 1.0 (no taper), also cap is created
StraightTube : 2 : dist : 1.0 : 1

Petal : < length > : < width > : < width bias > : < curl > : < detail >

Parametric object designed as a general purpose petal or leaf. This creates a petal shape (Surface) along the Turtle's heading (x axis). The petal is a general oval based on length and width. The width bias controls where the buldge of the oval is (0=beginning, 1 = tip). Curl controls how much the petal curls into itself - this is the overall value in radians. The detail will effect this final shape. Too little detail with a large curl will create segmented objects. A Petal Grasshopper file is distributed with the project so you can test parameters.

< length > = length of the petal
< width > = width of the petal
< width bias > = where width is applied to petal: 0=base, 0.5 = middle, 1 = end
< curl > = amount of curl in petal in radians (overall rotation)
< detail > = amount of detail in final petal surface, the more detail there is, the smoother the curled shape: min=3

#create a petal along the turtle's heading
# use variables len, wid, bias, and curl, with a constant detail of 6
Petal : len : wid : bias : curl : 6

SucculentLeaf : < length > : < width > : < thickness > : < edge thickness factor > : < curl > : < cup > : < width bias > : < loft type >

Parametric object designed as a general purpose leaf with thickness (inspired by a variety of succulents). This creates a lofted surface oriented along the Turtle's heading (x axis). The SucculentLeaf's shape is based on length and width and thickness. The edge thickness factor is the proportion of the edges to the overall thickness. The width bias controls where the buldge of the oval is (0=beginning, 1 = tip). Curl controls how much the petal curls along the x-axis (tilts upward with more curl), and Cup controls the curvature across the x-axis. A SucculentLeaf Grasshopper file is distributed with the project so you can test parameters.

< length > = length of the succulent leaf
< width > = width of the succulent leaf
< thickness > = thickness of the succulent leaf
< edge thickness factor > = multiplier against thickness that controls the thickness if the edge (0=sharp / pinched, 1=thick / round)
< curl > = amount of curl in petal along x-axis (-1 to 1, 0 = no curl / flat)
< cup > = amount of cup in petal across x-axis (-1 to 1, 0 = no cup / flat)
< width bias > = where width is applied to succulent leaf: 0=base, 0.5 = middle, 1 = end
< loft type > = integer value for loft type: normal = 0, loose = 1, tight = 2, straight = 3, developable = 4, uniform = 5

#create a succlent leaf along the turtle's heading
SucculentLeaf : l : w : t : 0.3 : curl : cup : 0.85 : L_LOOSE

Thorn : < length > : < width > : < height > : < height taper bias > : < width taper bias > : < curl > : < loft type >

Parametric object designed as a general purpose thorn. This creates a lofted surface oriented along the turtle's heading (x-axis). The thorn is based on length with a base defined by width and height in the YZ Plane. The thorn can be tapered along it's width and height (0=taper begins at base, 1=taper happens towards end). Curl controls the curvature along the x-axis. A Thorn Grasshopper file is distributed with the project so you can test parameters.

< length > = length of the thorn
< width > = width of the thorn's base
< height > = height of the thorn's base
< height taper bias > = where the height taper starts (0=begining, 1 = end)
< width taper bias > = where the width taper starts (0=begining, 1 = end)
< curl > = amount of curl in thorn along x-axis (-1 to 1, 0 = no curl / flat)
< loft type > = integer value for loft type: normal = 0, loose = 1, tight = 2, straight = 3, developable = 4, uniform = 5

#create a thorn pointing in the direction of the turtle's heading
Thorn : len : wid : hgt : hb : wb : crl : L_NORMAL

LowPolyLeaf : < length > : < width > : < width bias > : < curl >

Parametric object designed as a low polygon count leaf (4 polys). This creates a roughly oval mesh oriented along the turtle's heading (x-axis). Curl controls the curvature along the x-axis. Texture mapping is planar, so a lot of curl can distort mapping. A LowPolyLeaf Grasshopper file is distributed with the project so you can test parameters.

< length > = length of the leaf shape
< width > = width of the leaf shape
< width bias > = where the width occurs (0=begining, 0.5 = middle, 1 = end)
< curl > = amount of curl in the leaf (-1 to 1, 0 = no curl / flat)

#create a simple mesh leaf orientated along the turtle's heading
#with a width bias of 0.2, it will be pear-shaped
LowPolyLeaf : l_len : l_wid : 0.2 : 0.31
